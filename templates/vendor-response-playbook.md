## RESPONSE CLASSIFICATION SYSTEM

### Category 1: POSITIVE - INTERESTED
**Indicators:**
- "This sounds great"
- "I'd love to learn more"
- "Let's set up a call"
- Asks for more details/deck
- Provides pricing/availability

**Response Time:** <2 hours during business hours, <24h otherwise

**Response Template:**
```
[Name],

Fantastic! I'll send over the full partnership deck today with:
- Detailed event production timeline
- Venue photos and floor plans
- Sponsorship deliverable breakdown
- Past event examples and social reach metrics
- Attendee demographics

Are you available for a 15-minute call [Day] or [Day] afternoon to discuss next steps? I'm flexible on timing.

Looking forward to partnering with [Brand Name]!

Best,
Vinny
Venn Social Events
vinny@vennapp.co | 925-389-4794
```

**Follow-up Actions:**
1. Send full pitch deck (use event-specific template)
2. Add Google Calendar invite for proposed call times
3. Update CRM: status → 'interested', add lastInteractionAt
4. Set 48h reminder for follow-up if no response to deck
5. CC Zed if sponsorship value >$5K

### Category 2: BUDGET CONCERNS
**Indicators:**
- "That's outside our budget"
- "We don't have funds allocated"
- "Can you work with a smaller budget?"
- "What's the minimum investment?"

**Response Time:** <4 hours (shows flexibility, not desperation)

**Response Template - Partnership Alternatives:**
```
[Name],

Totally understand budget considerations. A few alternative partnership models that might work better:

1. **Product-in-kind:** [Brand] provides [product/service] in exchange for brand integration → we cover production costs → revenue share from ticket sales (you earn back value)

2. **Co-hosting model:** [Brand] and Venn co-promote the event → split costs and revenue 50/50 → shared upside, shared risk

3. **Reduced package:** $[Lower Amount] for core deliverables only:
   - [Core item 1 - e.g., bar tab/product]
   - [Core item 2 - e.g., basic branding]
   - No premium add-ons (photography, social campaigns, etc.)

4. **Trial partnership:** Sponsor one smaller event first ($[Smaller Amount], [Lower Capacity] guests) → prove ROI → scale to larger events

Which model interests you most? Happy to build a custom proposal around your budget constraints.

Best,
Vinny
```

**Follow-up Actions:**
1. Update CRM: notes → 'budget_concern', add budget feedback
2. Wait for their preferred model response
3. If no response in 72h → send brief check-in
4. Document preferred partnership models for future similar brands


5. Track no-response rate by category (spirits vs wine vs lifestyle) to identify pattern

**Max Follow-ups:** 2 (initial + 2 follow-ups = 3 total touches)

## EMAIL THREADING RULES (CRITICAL)

**ALWAYS reply in existing email thread:**
```bash
# Find the message ID from vendor's email
gog gmail messages search "from:vendor@example.com subject:inquiry" --max 1

# Reply in thread (REQUIRED)
gog gmail send --reply-to-message-id="MESSAGE_ID" \
  --reply-all \
  --subject="Re: [Original Subject]" \
  --body='[Response text]'
```

**NEVER create new thread when replying to vendor emails** - destroys professional communication flow.

## ESCALATION TRIGGERS

**When to CC Zed immediately:**
1. Sponsor commits $5K+ (needs final approval)
2. Brand requests contract/legal review
3. Venue wants deposit/commitment
4. Pricing negotiation reaches final stages
5. Brand asks questions you can't answer confidently

**When to handle independently:**
1. Initial positive responses (send deck)
2. Budget concerns (offer alternatives)
3. Timing discussions (schedule follow-up)
4. Basic venue/logistics questions
5. Referrals to different contacts

## COMMON MISTAKES TO AVOID

❌ **Don't:**
- Reply to Zed-forwarded vendor emails without authorization
- Mention budget in initial outreach (collect quotes first)
- Send duplicate emails to same organization
- Create new email thread when replying (always use --reply-to-message-id)
- Let vendor be last to respond (always close the loop)
- Update CRM "later" (same-minute updates required)

✅ **Do:**
- Check SENT folder before replying (avoid duplicate team responses)
- Verify event is not archived before ANY outreach
- Reply within 24h to keep momentum
- Provide clear next steps in every email
- Track all interactions in CRM immediately
- Leave door open even when declining/declined
